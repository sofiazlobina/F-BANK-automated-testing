# Отчёт о дефекте

**ID дефекта:** BUG‑001  
**Заголовок:** Система допускает перевод суммы 0 ₽  
**Приоритет:** Major  
**Серьёзность:** Critical  
**Окружение:** Edge, Windows 11, разрешение 1920×1080 
**Предусловия:** сервис F‑Bank запущен, доступен счёт «Рубли» с положительным балансом и резервом.

## Шаги воспроизведения
1. Открыть сервис F‑Bank.
2. Нажать на блок «Рубли» (счёт с балансом и резервом).
3. Ввести корректный 16‑значный номер карты (например, 1111 2222 3333 4444).
4. В поле суммы перевода ввести «0».
5. Нажать кнопку «Перевести».

## Фактический результат
Перевод на 0 ₽ принимается, появляется уведомление о принятии запроса.

## Ожидаемый результат
Система выдаёт ошибку с текстом «Невозможно перевести 0 ₽». Кнопка «Перевести» остаётся неактивной при вводе «0».

## Скриншоты/логи
* скриншот: `bug_001_zero_transfer.png`

## Комментарии
Дефект влияет на логику бизнес‑процесса — позволяет нецелевые операции. Пользователь может случайно отправить нулевой перевод, что создаёт лишнюю нагрузку на систему и может привести к путанице в истории операций.

# Отчёт о дефекте

**ID дефекта:** BUG‑002  
**Заголовок:** Пустое поле суммы трактуется как 0 ₽  
**Приоритет:** Major  
**Серьёзность:** Major  
**Окружение:** Edge, Windows 11, разрешение 1920×1080
**Предусловия:** сервис F‑Bank запущен, доступен счёт «Рубли».

## Шаги воспроизведения
1. Открыть сервис F‑Bank.
2. Нажать на блок «Рубли».
3. Ввести корректный номер карты.
4. Оставить поле суммы пустым.
5. Нажать кнопку «Перевести».

## Фактический результат
Система воспринимает пустое поле как «0 ₽», позволяет совершить перевод.

## Ожидаемый результат
Система показывает сообщение «Пожалуйста, введите сумму перевода». Кнопка «Перевести» неактивна при пустом поле.

## Скриншоты/логи
* скриншот: `bug_002_empty_amount.png`

## Комментарии
Критичный сценарий — пользователи могут случайно отправить «нулевой» перевод. Проблема UX: отсутствие явной валидации пустого поля.

# Отчёт о дефекте

**ID дефекта:** BUG‑003  
**Заголовок:** Система принимает 17‑значный номер карты  
**Приоритет:** Major  
**Серьёзность:** Major  
**Окружение:** Edge, Windows 11, разрешение 1920×1080 
**Предусловия:** сервис F‑Bank запущен, доступен счёт «Рубли».

## Шаги воспроизведения
1. Открыть сервис F‑Bank.
2. Нажать на блок «Рубли».
3. В поле для ввода номера карты ввести 17 цифр (например, 11112222333344445).
4. Проверить, принимается ли номер (активация поля, отсутствие ошибки).
5. Проверить, появляется ли кнопка «Перевести».

## Фактический результат
Система принимает 17‑значный номер, кнопка «Перевести» активна.

## Ожидаемый результат
Система выдаёт ошибку «Некорректная длина номера карты» (красное сообщение под полем ввода). Кнопка «Перевести» не активна.

## Скриншоты/логи
* скриншот: `bug_003_17_digits_card.png`

## Комментарии
Нарушение безопасности — некорректная валидация данных карты. Может привести к ошибкам при обработке платежа на стороне платёжного шлюза.

# Отчёт о дефекте

**ID дефекта:** BUG‑004  
**Заголовок:** Система допускает отрицательный перевод  
**Приоритет:** Critical  
**Серьёзность:** Blocker  
**Окружение:** Edge, Windows 11, разрешение 1920×1080 
**Предусловия:** сервис F‑Bank запущен, доступен счёт «Рубли».

## Шаги воспроизведения
1. Открыть сервис F‑Bank.
2. Нажать на блок «Рубли».
3. Ввести корректный номер карты.
4. В поле суммы ввести отрицательное значение (например, «–100»).
5. Нажать кнопку «Перевести».

## Фактический результат
Система принимает отрицательную сумму, рассчитывает комиссию (например, «–10 ₽» для суммы «–100 ₽»), позволяет совершить перевод.

## Ожидаемый результат
Система блокирует ввод отрицательного числа, показывает ошибку «Сумма перевода должна быть положительной». Кнопка «Перевести» неактивна.

## Скриншоты/логи
* скриншот: `bug_004_negative_amount.png`

## Комментарии
Критический баг — нарушает финансовую логику, возможен несанкционированный возврат средств. Прямой риск финансовых потерь для банка.

# Отчёт о дефекте

**ID дефекта:** BUG‑005  
**Заголовок:** Некорректный расчёт комиссии (округление до десятков)  
**Приоритет:** Major  
**Серьёзность:** Major  
**Окружение:** Edge, Windows 11, разрешение 1920×1080  
**Предусловия:** сервис F‑Bank запущен, доступен счёт «Рубли».

## Шаги воспроизведения
1. Открыть сервис F‑Bank.
2. Нажать на блок «Рубли».
3. Ввести корректный номер карты.
4. Ввести сумму перевода 99 ₽.
5. Проверить расчёт комиссии (должна быть 10 % от суммы, округлённая вниз).

## Фактический результат
Комиссия рассчитывается как 10 ₽ (округление до десятков), итоговая сумма = 0 ₽.

## Ожидаемый результат
Комиссия = 9 ₽ (10 % от 99 ₽, округлено вниз). Итоговая сумма с комиссией = 108 ₽.

## Скриншоты/логи
* скриншот: `bug_005_commission_rounding.png`

## Комментарии
Влияет на точность финансовых расчётов, может привести к недополучению комиссии банком. Ошибка округления нарушает требования бизнес‑логики.
