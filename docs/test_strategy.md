# Стратегия ручного тестирования F‑Bank

**Версия:** 1.0  
**Дата:** 2026‑04‑25

## 1. Цели тестирования
* проверить корректность перевода средств между счетами;
* убедиться в валидности входных данных (номер карты, сумма);
* подтвердить правильность расчёта комиссии (10 %, округление вниз);
* выявить дефекты в UI/UX интерфейса перевода.

## 2. Объём тестирования
**В scope:**
* переводы в рублях с основного счёта;
* валидация номера карты (16 цифр);
* валидация суммы перевода (положительная, не пустая, не 0 ₽);
* расчёт комиссии и итоговой суммы.

**Out of scope:**
* валютные переводы;
* переводы между разными пользователями;
* интеграция с внешними платёжными системами.


## 3. Подходы и техники
* эквивалентное разбиение (валидные/невалидные суммы);
* анализ граничных значений (0 ₽, максимальная сумма, отрицательная сумма);
* проверка сообщений об ошибках;
* тестирование UI‑элементов (активность кнопок, подсказки).

## 4. Критерии приёмки
* все тест‑кейсы пройдены (статус Pass);
* критические дефекты исправлены (Critical/Major);
* покрытие требований ≥ 90 %.

## 5. Ресурсы
* тестировщик: 1 чел.;
* окружение: локальный запуск F‑Bank (`http://localhost:8000/`);
* инструменты: Edge.

## 6. Риски
* недоступность тестового окружения;
* изменение требований в процессе тестирования;
* неполнота тестовых данных.
