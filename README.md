# README для проекта «F-BANK: автоматизированное тестирование»

## 1. Общее описание проекта

Проект посвящён автоматизированному тестированию функционала банковского веб‑приложения «F‑BANK». Основная задача — проверка корректности работы:
* модулей перевода средств;
* расчёта комиссии;
* валидации входных данных.

Тестирование реализовано с помощью:
* **pytest** — фреймворк для написания и запуска тестов;
* **Selenium WebDriver** — автоматизация взаимодействия с веб‑интерфейсом.

## 3. Функционал, покрытый тестами

Проверяемые сценарии:
* валидация номера банковской карты (длина, формат);
* проверка суммы перевода (отрицательные значения, ноль, пустые поля);
* расчёт комиссии за перевод (корректность вычислений, округление);
* ограничения по балансу и зарезервированным средствам;
* обработка ошибок (алертные сообщения, валидационные ошибки).

## 4. Используемые технологии и инструменты

* **Python 3.x** — основной язык разработки;
* **pytest** — фреймворк для тестирования;
* **Selenium WebDriver** — взаимодействие с браузером;
* **MS Edge Driver** (`msedgedriver.exe`) — драйвер для Microsoft Edge;
* **logging** — логирование событий;
* **JSON** — формат хранения тестовых данных;
* **Markdown** — формат документации и отчётов.

## 5. Инструкция по запуску

### Шаг 1. Установка зависимостей

Выполните в терминале:
```bash
pip install -r requirements.txt
```

### Шаг 2. Настройка окружения
1. Убедитесь, что в папке config/ присутствует файл config.py с корректными настройками:

    1. URL приложения для тестирования — укажите актуальный URL тестового стенда;

    2. параметры подключения к БД (если требуется) — хост, порт, имя базы данных, учётные данные;
    
    3. настройки логирования — уровень логирования (DEBUG, INFO, WARNING), путь к файлу логов.

2. Проверьте наличие драйвера msedgedriver.exe в корне проекта — он должен соответствовать версии вашего браузера Microsoft Edge.

3. При необходимости обновите путь к драйверу в config.py, указав корректное расположение файла.

### Шаг 3. Запуск тестов

Запуск всех тестов:
```
pytest automated_tests/tests/
```

Запуск отдельного теста:
``` 
pytest automated_tests/tests/test_transfer_invalid.py
```
